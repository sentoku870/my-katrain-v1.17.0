# Ideas（アイデア・構想）

このフォルダには、将来の機能拡張に関するアイデアや構想メモが格納されています。

## 収録ファイル

| ファイル | 概要 |
|----------|------|
| `案１ ペース配分＆ティルト検知「Pacing & Tilt Doctor」.md` | 対局中のペース配分とメンタル状態の検知機能 |
| `案２ ポジティブ・フィードバック生成機能「My Style Identity」.md` | プレイヤーの棋風を分析しポジティブなフィードバックを生成 |
| `案３ 形勢判断・リスク管理チェッカー「Risk Reward Alignments」.md` | 形勢判断とリスク・リターンのバランス分析 |
| `案４ 弱点克服・反復ドリル「Weakness Repeater」.md` | 弱点パターンを繰り返し練習するドリル機能 |
| `案５ 名局選定と学習ガイド生成「Smart Kifu Curator」.md` | 学習に適した棋譜を選定しガイドを生成 |

## ステータス

これらは構想段階のアイデアです。実装の優先度は未定です。

## 関連フォルダ

- `docs/future/` - Phase 52時点で延期された仕様書（より具体的な設計）
- `docs/archive/specs-implemented/` - 実装済みの仕様書

1. アイデアを詳細な仕様書に落とし込む
2. 仕様書を `docs/` 直下に配置
3. Phaseとして実装計画を立てる
4. 実装後、仕様書を `docs/archive/specs-implemented/` に移動
